# ManualDownloadEngine

An orchestrator wrapper engine designed to manually inject, catalog, and process supplemental, ad-hoc DigitalCoast digital elevation models (DEMs) into the downstream model pipelines.

## 💡 Overview

The `ManualDownloadEngine` acts as a central control panel. When your team has to manually grab extra datasets that weren't caught in the automated sync phases, this engine passes those files through all the sub-engines we just documented. It ensures manual overrides cleanly integrate into the master virtual mosaics, spatial grids, and training masks without breaking a sweat.

Instead of reinventing the wheel, it acts as a composite wrapper over 4 specialized core subsystems:
1. **`RasterVRTS3Engine`** - To group and structure raw data files into a master virtual raster index.
2. **`RasterMaskS3Engine`** - To construct updated classification training masks.
3. **`GridDigitalCoastEngine`** - To clip the new data parameters down to standardized tile footprints.
4. **`MetadataS3Engine`** - To isolate and catalog chronological timestamps.

---

## ⚙️ Core Architecture & State

* **Base Class:** Operates as a standalone master orchestrator script.
* **Workspace Isolation:** Automatically manages previous run telemetry. If an old `log_prints.txt` file exists in the outputs directory, it backs it up immediately with an updated epoch timestamp flag to prevent rolling overwrite data loss.
* **Hardcoded Constraints:** Currently explicitly targets the **`ER_3`** (EcoRegion 3) geographic workspace and routes operations using a cloud execution environment context (`env='aws'`).

## 🛠️ Execution Sequential Order

When executing `engine.run()`, order is highly dependent. Modifying the execution chronology will result in downstream pipeline failures.

<Sequence>
{/* Reason: Sub-engines have rigid data dependencies; you cannot clip tiles or build training masks until the parent virtual mosaics (VRTs) are structured. */}
  <Step title="VRT Index Initialization" subtitle="process_vrt()">

    Cross-intersects the generated virtual indexes against the master spatial index grid and cuts the datasets down to standardized BlueTopo layout cells.
  </Step>
</Sequence>

## 🛠️ Method Reference

### `run()`
* **What it does:** Primary runner function evaluating workspace integrity and managing engine execution steps.
* **The Specifics:** Checks for existing log entries, renames active documents if conflicts are found by appending an active epoch timer flag (`log_prints_[timestamp].txt`), and fires sub-modules sequentially.

### `process_vrt()`
* **What it does:** Assembles loose input rasters into structural virtual indices.
* **The Specifics:** Spins up a localized Dask computing cluster over cloud parameters, locates manually downloaded asset keys, matches configuration signatures (`NCMP` criteria), structures them into separate spatial bins, and closes cluster worker threads upon completion.

### `process_masks()`
* **What it does:** Incorporates supplemental parameters into target classification training layers.
* **The Specifics:** Targets your output directories and forces execution branches to incorporate manual override subfolders to compile categorical training pixels.

### `process_grid_tiles()`
* **What it does:** Cookie-cuts raw data indices down to match operational grid boundaries.
* **The Specifics:** Sets `manual_download=True` inside the underlying sub-engine to route grid calculations straight through custom asset pathways.

### `process_metadata()`
* **What it does:** Chronologically maps out metadata indices for loose manual overrides.
* **Gotchas / Notes:** *Currently bypassed in the main execution script loop.* When called, it explicitly targets the `DigitalCoast_manual_downloads` bucket directories to read structural JSON properties and build unified timeline trackers.

### `rebuild_training_mask()`
* **What it does:** Direct standalone validation mask repair utility.
* **Gotchas / Notes:** *Currently commented out in production code.* Can be triggered if processing boundaries are interrupted, using pre-computed, localized partial mask images on disk to remerge data matrices.

## 🚀 Quick Start / Usage Example

```python
import pathlib
from hydro_health.helpers.tools import Param
from hydro_health.engines.ManualDownloadEngine import ManualDownloadEngine

# 1. Establish path roots targeting output structures
OUTPUTS = pathlib.Path("/var/data/hydro_health/outputs")

# 2. Build parameter configurations targeting the cloud runtime environment
param_lookup = {
    'output_directory': Param(str(OUTPUTS)),
    'env': 'aws'
}

# 3. Instantiate the master manual ingestion compiler
manual_pipeline = ManualDownloadEngine(param_lookup=param_lookup)

# 4. Trigger the sequential processing workflow
manual_pipeline.run()
